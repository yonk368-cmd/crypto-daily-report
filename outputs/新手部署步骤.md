# Crypto Daily Report V3-lite 新手部署步骤

## 你已经有的文件

项目包：

`crypto_daily_report_v3_lite.zip`

这个包已经包含：

- `crypto_daily_report.py`
- `requirements.txt`
- `README.md`
- `.github/workflows/crypto_daily_report.yml`
- `reports/.gitkeep`

## 推荐方式：用 GitHub Desktop 上传

这是最适合新手的方式，因为它会自动处理隐藏的 `.github` 文件夹。

### 第 1 步：安装 GitHub Desktop

打开：

https://desktop.github.com/

下载安装并登录你的 GitHub 账号。

### 第 2 步：解压项目包

把 `crypto_daily_report_v3_lite.zip` 解压成一个文件夹。

建议文件夹名字：

`crypto-daily-report`

### 第 3 步：用 GitHub Desktop 创建仓库

打开 GitHub Desktop：

1. 点 `File`
2. 点 `Add Local Repository...`
3. 选择刚才解压出来的 `crypto-daily-report` 文件夹
4. 如果提示不是 Git 仓库，点 `create a repository`
5. Repository name 填：`crypto-daily-report`
6. Local path 选这个项目文件夹
7. 点 `Create Repository`

### 第 4 步：发布到 GitHub

在 GitHub Desktop 顶部点：

`Publish repository`

建议：

- Name：`crypto-daily-report`
- 勾选或选择 `Keep this code private`

然后点发布。

### 第 5 步：配置 Gmail Secrets

进入 GitHub 网页里的这个仓库：

`Settings -> Secrets and variables -> Actions -> New repository secret`

添加 3 个 Secret：

1. `GMAIL_USER`
   - 值：你的 Gmail 地址

2. `GMAIL_APP_PASSWORD`
   - 值：Google 账号里生成的 16 位 App Password
   - 注意：不是 Gmail 登录密码

3. `REPORT_TO_EMAIL`
   - 值：你希望接收日报的邮箱，可以和 Gmail 地址相同

### 第 6 步：手动测试一次

进入仓库：

`Actions -> Crypto Daily Report -> Run workflow`

运行成功后，你应该看到：

- Gmail 收到日报
- 仓库出现 `reports/YYYY-MM-DD.md`

## Gmail App Password 怎么生成

1. 打开 Google Account：
   https://myaccount.google.com/
2. 进入 `Security`
3. 开启 `2-Step Verification`
4. 搜索或进入 `App passwords`
5. 创建一个 app password
6. 复制生成的 16 位密码
7. 填入 GitHub Secret：`GMAIL_APP_PASSWORD`

## 可选：ETH ETF 手动覆盖

如果 Farside 自动抓取失败，日报会显示 ETF 数据待验证。

你也可以在 GitHub 里配置：

`Settings -> Secrets and variables -> Actions -> Variables`

添加：

- `ETH_ETF_DATE`，例如：`01 Jul 2026`
- `ETH_ETF_TOTAL_USD_M`，例如：`148.5` 或 `(32.1)`

这样日报会使用你手动填的 ETF 数据。
